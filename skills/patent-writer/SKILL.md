# 专利撰写 Skill

专业专利申请文件自动撰写工具，支持根据技术资料生成完整的专利申请文档。

## 工作流程

### 第一步：收集和分析输入材料

1. **读取用户提供的文件**
   - 技术交底书（推荐）
   - 发明创意描述文档
   - 技术方案文档
   - 研发报告
   - 产品说明书

2. **提取关键信息**
   - 发明名称和技术领域
   - 要解决的技术问题
   - 核心技术方案和创新点
   - 主要技术特征
   - 技术效果和优势
   - 具体实施例和参数

3. **信息不足时主动询问**
   - 背景技术和现有方案
   - 具体实施细节
   - 参数范围和数据
   - 应用场景

### 第二步：规划专利结构

根据 `references/patent-structure.md` 确定专利结构：

```
专利申请书
├── 请求书（发明名称、申请人信息）
├── 权利要求书（独立权利要求 + 从属权利要求）
├── 说明书
│   ├── 技术领域
│   ├── 背景技术
│   ├── 发明内容
│   ├── 附图说明
│   └── 具体实施方式
├── 摘要
└── 附图
```

### 第三步：撰写各组成部分

#### 3.1 权利要求书撰写

参考 `references/claims-guide.md`：

1. **确定独立权利要求**
   - 识别全部必要技术特征
   - 使用上位概念扩展保护范围
   - 采用标准格式：主题 + 前序部分 + 特征部分

2. **构建从属权利要求**
   - 添加非必要技术特征
   - 形成递进式保护层次
   - 规范引用格式

```
权利要求1（独立）：
一种[主题]，其特征在于，包括：
[必要特征1]；
[必要特征2]。

权利要求2（从属）：
根据权利要求1所述的[主题]，其特征在于，[附加特征]。
```

#### 3.2 说明书撰写

参考 `references/specification-guide.md`：

| 部分 | 要点 |
|------|------|
| 技术领域 | 一句话明确技术范畴 |
| 背景技术 | 现有技术 + 存在问题 |
| 发明内容 | 技术问题 + 方案 + 有益效果 |
| 附图说明 | 图号 + 名称 + 部件编号 |
| 具体实施方式 | 详细实施例 + 参数 + 效果数据 |

#### 3.3 摘要撰写

- 概括发明主题和技术特征
- 说明主要技术效果
- 不超过300字
- 指定摘要附图

### 第四步：输出完整专利文件

使用 `assets/patent-template.md` 格式输出：

```markdown
# 专利申请书

## 请求书
[发明名称、申请人信息]

## 权利要求书
[权利要求1-N]

## 说明书
### [发明名称]
### 技术领域
[内容]
### 背景技术
[内容]
### 发明内容
[内容]
### 附图说明
[内容]
### 具体实施方式
[内容]

## 摘要
[内容]
```

## 质量检查清单

撰写完成后，逐项检查：

### 权利要求书
- [ ] 独立权利要求包含全部必要技术特征
- [ ] 使用上位概念扩展保护范围
- [ ] 从属权利要求引用格式正确
- [ ] 无累积引用（"和"应改为"或"）

### 说明书
- [ ] 背景技术问题与发明内容对应
- [ ] 技术方案描述完整清楚
- [ ] 包含具体实施例
- [ ] 包含参数范围（如适用）
- [ ] 包含效果验证数据（如适用）

### 摘要
- [ ] 不超过300字
- [ ] 概括主要技术特征
- [ ] 说明技术效果

## 专利类型选择

| 类型 | 适用场景 | 审查要求 |
|------|----------|----------|
| 实用新型 | 产品结构改进、格式要求较低 | 较低，适合初学者 |
| 发明 | 方法创新、技术突破、高价值专利 | 较高，建议专业代理 |

## 使用示例

**示例1：用户提供技术交底书**
```
用户：请根据这份技术交底书帮我撰写专利
[上传技术交底书文件]

助手：
1. 读取并分析技术交底书
2. 提取核心发明点和技术特征
3. 按流程生成完整专利申请文件
```

**示例2：用户描述发明创意**
```
用户：我有一个智能温控系统的发明想法...

助手：
1. 询问技术细节（如信息不足）
2. 整理技术方案
3. 生成专利申请文件
```

## 注意事项

1. **保护范围**：独立权利要求不宜过窄，避免非必要限制
2. **术语规范**：使用标准技术术语，保持全文一致
3. **充分公开**：具体实施方式需足够详细，使本领域技术人员能够实现
4. **2025新规**：强调实际解决效果，需提供实施例、参数、验证数据
