# 案例分析答题方法

做案例分析辅导、讲真题案例、模拟出题时加载。

## 一、概述
- 5 道大题，每题约 15 分，满分 75，**45 分及格**
- 与综合知识连考（共 4 小时），建议**留 2.5 小时**给案例
- 考点围绕 **IT 服务全生命周期**（教材 Ch.12 核心延伸）
- 题型：概念默写、判断改错、简答、计算、场景分析

## 二、答题总原则
1. **分点作答**：用 ①②③ 编号，阅卷按点给分
2. **先写结论后展开**：判断题先给"正确/错误"，再写理由
3. **用教材术语**：答 PPTR、SLA、PDCA 等标准词，不用大白话
4. **改错题三步**：指出错误 → 给出正确表述 → 必要时补依据
5. **计算题写过程**：列公式→代数→结果，过程分不能丢
6. **宁多勿少**：不会的题把相关知识点都写上，按点得分

## 三、五大题型套路

### 1. 概念默写题（最易拿分）
考 PPTR 四要素、服务目录、SLA、风险管理等定义性内容。对策：**背熟核心清单**（见下"高频默写点"），原文默写。

### 2. 判断改错题
给出一段描述，判断对错并改正。对策：逐句扫描，找**特征错位、张冠李戴**（如把"人员要素"内容说成"资源要素"）。

### 3. 场景分析题
给一个 IT 服务场景（如某公司运维出问题），问原因/对策。对策：套 **PPTR 四要素 + 生命周期阶段**框架排查，分点作答。

### 4. 计算题（必考）
常见计算见下"计算公式集"。

### 5. 填空题
教材原文挖空，考记忆准确性。

## 四、高频默写点（务必背熟）

**PPTR 四要素各自包含**：
- **人员要素 People**：人员储备、岗位结构与职责、绩效、培训、知识技能管理、团队建设
- **过程要素 Process**：服务报告、事件管理、问题管理、变更管理、配置管理、发布管理
- **技术要素 Technology**：技术研发规划、发现问题技术、解决问题技术、技术手册
- **资源要素 Resource**：服务台、备件库、知识库、运维工具、数据/信息资源

**服务目录**：分**业务服务目录**（面向客户）和**技术服务目录**（内部支撑）。含服务名称、描述、分类、状态、响应/解决时间、负责人等。

**SLA 三类协议**：
- **SLA** 服务级别协议：客户 ↔ 服务提供商（外部）
- **OLA** 运营级别协议：服务团队内部（内部支撑）
- **UC** 支持合同：与外部供应商

**IT 服务风险管理**：风险识别 → 风险分析（定性/定量）→ 风险控制（规避/转移/降低/接受）→ 风险监控

**IT 服务持续改进 PDCA**：Plan 策划 → Do 实施 → Check 检查 → Act 处置

## 五、计算公式集

### 可用性 / 可靠性（高频）
- **MTBF** 平均无故障时间 = 系统正常运行总时间 ÷ 故障次数
- **MTTR** 平均修复时间 = 故障修复总时间 ÷ 故障次数
- **MTBSI** 平均系统间隔时间 = MTBF + MTTR
- **可用性 A** = MTBF ÷ (MTBF + MTTR) × 100%

### SLA 达成率
- SLA 达成率 = 达成 SLA 的事件数 ÷ 总事件数 × 100%
- 常见 SLA 等级：响应时间、解决时间、可用性百分比（如 99.9%）

### 人员配置
- 所需人数 = 总工作量(人时) ÷ 〔有效工时 × 出勤率 × 利用率〕
- 注意扣除非工作时间、培训、休假

### 服务成本
- 总成本 = 直接成本(人力/资源) + 间接成本(管理/分摊)
- 单位服务成本 = 总成本 ÷ 服务量

## 六、IT 服务全生命周期命题主线
```
规划设计 → 部署实施 → 服务运营 → 持续改进 → 监督管理
   ↓          ↓          ↓          ↓          ↓
 PPTR设计   资源/过程   日常运行    PDCA       风险/质量
 SLA/目录   部署上线   事件/问题   测量回顾   考核评价
```
案例题几乎都落在这条主线的某个环节，**先定位环节，再套 PPTR 或对应知识点答题**。

## 七、得分技巧
- 改错题每处错误通常对应一个知识点，**改对一处得一分**，多找多得分
- 计算题单位、小数位数要规范（百分比保留 1-2 位）
- 默写题用**短句+术语**，不要长篇大论，方便阅卷找点
- 时间紧时，先做计算和默写（确定性高），场景题最后写
