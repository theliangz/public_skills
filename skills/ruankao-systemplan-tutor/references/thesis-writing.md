# 论文写作方法

做论文辅导、批改、出题、提供写作框架时加载。输出论文成品时配合 `assets/thesis-template.md`。

## 一、基本要求（2025 改版后）
- **1 道题**（近年直接给题，非四选一），**不写摘要**
- 正文字数 **2000–2500 字**（不足 2000 难以及格）
- 满分 75，**45 分及格**，单独场次约 2 小时
- 围绕 **IT 服务管理**展开，**必须结合真实项目实例**

## 二、评分维度（阅卷给分点）
1. **切题**：紧扣题目论点，不跑题（最高权重）
2. **结合实际**：有具体项目、真实数据、个人角色
3. **结构完整**：背景 + 正文 + 结尾三段式
4. **理论支撑**：用教材概念（PPTR、SLA、PDCA、ITSS）支撑实践
5. **字数达标**：2000–2500
6. **逻辑表达**：分点清晰、条理通顺

## 三、标准结构与字数分配（不写摘要）

| 段落 | 字数 | 内容要点 |
|------|------|---------|
| **一、项目背景** | 400–500 | 项目名称、时间、规模、自己的角色与职责、项目特点、为何涉及该论点 |
| **二、实践过程（主体）** | 1500–1700 | 围绕论点展开 3-5 个方面，每方面 = 理论 + 做法 + 效果 |
| **三、结尾总结** | 300 | 项目成果、个人收获/体会、不足与改进方向 |

## 四、常见出题方向（按 IT 服务全生命周期）
1. **IT 服务规划设计**（PPTR 四要素设计、服务目录、SLA 设计）
2. **IT 服务运营管理**（事件/问题/变更管理、人员与资源运营）
3. **IT 服务持续改进**（PDCA、服务测量与回顾）
4. **IT 服务监督管理**（风险管理、质量管理、考核评价）
5. **IT 服务团队/人员管理**（团队建设、绩效、培训）
6. **IT 服务营销/客户关系管理**

## 五、写作要点

### 项目背景段
- 交代清楚：什么项目、多大体量（用户数/设备数/系统数）、什么时间、你担任什么角色
- 点出项目面临的挑战，自然引出论点
- **切忌**：编造过于完美或过于宏大的项目；背景写成流水账

### 实践过程段（核心得分区）
- 围绕论点拆 **3–5 个子论点**，每段一个
- 每个子论点结构：**理论概念（教材术语）→ 具体做法（结合项目）→ 实施效果（数据/结果）**
- 举例要具体：用了什么工具、制定了什么指标、解决了什么问题、达到什么数值
- **切忌**：纯背理论；只有做法不提理论；只讲理论不结合项目

### 结尾段
- 总结项目成效（量化：可用性提升到 99.9%、故障率下降等）
- 谈个人体会与成长
- 提不足与未来改进（体现反思）
- **切忌**：草草收尾、不总结

## 六、避坑指南（高频失分点）
| 失分点 | 对策 |
|--------|------|
| 字数不足 2000 | 主体段写足 3-5 个方面，每方面 300+ 字 |
| 背景过长挤占正文 | 背景控制在 500 字内 |
| 空泛无项目 | 必须有项目名称、数据、个人角色 |
| 偏题/堆砌无关理论 | 每段首句回扣论点 |
| 纯理论无实践 | 理论与做法 1:1 配比 |
| 写成摘要式概括 | 展开细节，写"怎么做"而非"是什么" |

## 七、练习方法
1. **准备 1-2 个通用项目背景**（如某企业 IT 运维服务项目、某政务云运维项目），可适配多个论点
2. 按六大方向**各准备一篇**框架，填入项目
3. 限时 2 小时手写/机打完整成文
4. 对照评分维度自评，重点查字数与"理论+实践"配比

> 批改用户论文时：逐段检查字数、论点贴合度、理论实践配比、有无具体数据，按六维度给分并指出改进点。
